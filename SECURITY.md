# Security Policy

## Reporting a vulnerability

Please do not open a public issue for a suspected security vulnerability.

Use GitHub's **Report a vulnerability** option on the repository Security tab. Include the affected page or API route, reproduction steps, expected impact, and any suggested mitigation.

Reports will be reviewed as soon as practical. Please allow time for a fix before sharing details publicly.

## Supported version

Security fixes are applied to the current production version at [ld.fluz-tools.com](https://ld.fluz-tools.com).
